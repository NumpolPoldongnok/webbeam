# WebBeam

ส่งไฟล์ระหว่างเครื่องที่อยู่ **LAN เดียวกัน** แบบ peer-to-peer ผ่าน WebRTC
ทำงาน **offline** (ไม่ต้องมี internet) · **ไม่ต้องติดตั้ง** · **ไม่ผ่านเซิร์ฟเวอร์** · เป็น HTML ไฟล์เดียว

> ✅ ทดสอบจริงแล้วด้วย Chrome 2 instance: เชื่อมต่อ WebRTC + ส่งไฟล์ 5MB และ 120MB
> ครบถ้วน SHA-256 ตรงกัน (ดู [test/](test/))

## ใช้งาน

เปิด `index.html` ในเบราว์เซอร์ทั้งสองเครื่อง (ดับเบิลคลิกไฟล์ หรือเปิดจาก URL ที่โฮสต์ไว้)

1. **เครื่อง A**: เลือก "เริ่มการเชื่อมต่อ" → กด "สร้างรหัสของฉัน" → คัดลอกรหัส ส่งให้เครื่อง B
2. **เครื่อง B**: เลือก "เข้าร่วม" → วางรหัสจาก A → กด "สร้างรหัสตอบกลับ" → คัดลอก ส่งกลับให้ A
3. **เครื่อง A**: วางรหัสตอบกลับ → กด "เชื่อมต่อ"
4. เชื่อมต่อแล้ว → **ลากไฟล์มาวาง** ส่งได้ทั้งสองทาง

> รหัสที่แลกกันทำได้ทุกช่องทาง (แชต/โน้ต/พิมพ์มือ) แลกแค่ครั้งเดียวตอนเริ่ม

## ไฟล์ใหญ่ / โฟลเดอร์ปลายทาง

- ใช้ **Chrome / Edge** แล้วกด "เลือกโฟลเดอร์…" ฝั่งรับ → ไฟล์จะ **stream ลงดิสก์โดยตรง** รับหลาย GB ได้
- **Safari** ไม่รองรับเลือกโฟลเดอร์ → ไฟล์จะถูกดาวน์โหลดแทน และจำกัดขนาดตามหน่วยความจำ
  → แนะนำใช้ Chrome/Edge ทั้งสองเครื่อง

## เงื่อนไข

- ทั้งสองเครื่องต้องอยู่ **Wi-Fi/LAN วงเดียวกัน** (ใช้ host/mDNS candidates ต่อตรง ไม่ต้องมี STUN/internet)
- ถ้าเครือข่ายมี **AP/client isolation** เปิดอยู่ (Wi-Fi สาธารณะบางที่) เครื่องจะต่อกันไม่ได้

## โฮสต์เป็น landing page (ทางเลือก)

เป็น static ล้วน วางที่ไหนก็ได้:

```bash
# GitHub Pages
git init && git add . && git commit -m "WebBeam"
# push ขึ้น repo แล้วเปิด Settings → Pages → branch main
```

หรือเสิร์ฟในเครื่องเพื่อทดสอบ:
```bash
cd WebBeam && python3 -m http.server 8080
# เปิด http://localhost:8080
```

> หมายเหตุ: `clipboard.writeText` และ `showDirectoryPicker` ต้องการ **https** หรือ **localhost/file://**
> ถ้าโฮสต์เป็น http ธรรมดาในวง LAN ปุ่มคัดลอกอาจใช้ไม่ได้ (ยังเลือก-คัดลอกเองได้)
