# WebBeam

**แชต + ซิงค์โฟลเดอร์** ระหว่าง 2 เครื่องบน **LAN เดียวกัน** แบบ peer-to-peer (WebRTC) —
ทั้งสองอย่างใช้ **connection เดียวกัน ทำงานพร้อมกันได้** · offline · ไม่ต้องติดตั้ง · ไม่ผ่านเซิร์ฟเวอร์

แอปรวมอยู่ที่ [sync.html](sync.html) (มีแท็บ 💬 แชต / 🔄 ซิงค์) — `index.html` จะ redirect มาที่นี่

แชต + ส่งโค้ด/ไฟล์ · ประวัติเก็บในเครื่อง (local)

> ✅ ทดสอบจริงด้วย Chrome 2 instance: เชื่อมต่อ + แชต + ส่ง source file (โชว์ inline) + history persist หลัง reload (ดู [test/](test/))

## ใช้งาน

เปิดในเบราว์เซอร์ทั้งสองเครื่อง (แนะนำ Chrome/Edge)

1. ตั้ง **ชื่อของคุณ** → เลือกบทบาท **A (เริ่ม)** / **B (เข้าร่วม)**
2. แลกรหัสเชื่อมต่อกัน 1 รอบ (คัดลอก-วาง หรือ **สแกน QR**) → กดเชื่อมต่อ
3. จุดสถานะเป็น 🟢 → แผงเชื่อมต่อจะพับเอง เริ่มคุยได้เลย

## ฟีเจอร์

- **แชตข้อความ** — Enter ส่ง, Shift+Enter ขึ้นบรรทัด
- **ส่งเป็นบล็อกโค้ด** — ปุ่ม `</> โค้ด` (โชว์ monospace + ปุ่มคัดลอก)
- **ส่งไฟล์** — ปุ่ม 📎 หรือ **ลากมาวางในแชต**
  - **source file** (.swift/.js/.py/… ≤256KB) → โชว์เนื้อหา inline + คัดลอก/บันทึก
  - ไฟล์อื่น → การ์ดไฟล์ + ปุ่มบันทึก (ดาวน์โหลด)
- **ประวัติแชต (local)** — เก็บใน `localStorage` ของเบราว์เซอร์ (แต่ละเครื่องเก็บของตัวเอง) · เปิดใหม่ก็ยังอยู่ · ปุ่ม 🗑 ล้างได้

## 🔄 โหมดซิงค์โฟลเดอร์ ([sync.html](sync.html))

ซิงค์โฟลเดอร์ **สองทาง** ระหว่าง 2 เครื่องบน LAN (P2P, offline) — เหมาะกับโปรเจกต์ source code

- 🔥 **โหมด Hotspot — เชื่อมอัตโนมัติ ไม่ต้องแลกรหัส** (แนะนำเมื่ออยู่ hotspot/LAN เดียวกัน): รัน broker จิ๋วบน**เครื่องหลักเครื่องเดียว** ทั้งสองเครื่องจับคู่กันเอง · **หลุดแล้วต่อกลับเองอัตโนมัติ** ไม่ต้องแลกรหัสซ้ำ — ดูหัวข้อ "โหมด Hotspot" ด้านล่าง
- **เชื่อมแบบ manual** (ไม่ต้องมี broker): เลือกบทบาท A/B → แลกรหัส (คัดลอก-วาง หรือ **สแกน QR**) → **วางรหัสแล้วเชื่อมต่อให้อัตโนมัติ** ไม่ต้องกดปุ่มเพิ่ม · รหัสเป็นฟอร์แมตย่อ (gzip) สั้นลง ~3 เท่า สแกน QR ติดง่ายขึ้น
- **เครือข่ายบล็อก mDNS แล้วต่อไม่ติด?** ใส่ **"IP เครื่องนี้ (LAN)"** ทั้งสองเครื่อง (ดูจาก System Settings → Wi-Fi → Details) แล้วสร้างรหัสใหม่ — ระบบจะใช้ IP จริงแทน `.local` ทำให้ต่อตรงได้แม้ mDNS ถูกบล็อก (Wi-Fi คอนโด/ออฟฟิศ)
- หลุดการเชื่อมต่อ → ฝั่ง A **สร้างรหัสใหม่รอไว้ให้เอง** แลกรหัสรอบใหม่ได้ทันที
- **จำโฟลเดอร์ล่าสุด**: เปิดหน้าใหม่มีปุ่ม "📁 ใช้โฟลเดอร์ล่าสุด" กดครั้งเดียว (ถ้า Chrome จำ permission ไว้ จะเปิดให้อัตโนมัติเลย) — ไม่ต้องไล่หาโฟลเดอร์ทุกครั้ง
- **ทางเดียว** (เครื่องที่กด = ฝ่ายรับ) · กดทีไร = **คัดลอกทุกไฟล์ในโฟลเดอร์มาเขียนทับ** (ตรงไปตรงมา ไม่เดาว่าไฟล์ไหนเปลี่ยน) · รองรับโฟลเดอร์ย่อย
- **เร็วขึ้นมาก**: ไฟล์เล็ก (≤256KB) ถูก**มัดเป็นชุด** (สูงสุด 200 ไฟล์/8MB ต่อชุด) แล้ว gzip ทีเดียว → ลด overhead ต่อไฟล์ เหมาะกับ source tree ที่มีไฟล์เล็กเยอะ · ไฟล์ใหญ่ส่งเดี่ยวพร้อม progress · อัปเดต progress แบบ throttle ไม่ถ่วงการส่ง
- **ไฟล์ที่อีกเครื่องไม่มี จะไม่ถูกลบ** (เขียนทับเฉพาะที่ส่งมา ไม่ใช่ mirror) · ข้าม `.DS_Store`, `.git`, `node_modules`
- ✅ ทดสอบ e2e จริง (in-memory store บน WebRTC) + protocol test (`node test/protocol-vm-test.mjs` — ไม่ต้องมี Chrome): ดึงทั้งหมด/ชุดไฟล์, ดึงซ้ำคัดลอกครบทุกครั้ง, ทางเดียว, รหัสย่อ, IP แทน mDNS
- ข้อจำกัด: **Chrome/Edge เท่านั้น**

### 🔥 โหมด Hotspot — เชื่อมอัตโนมัติ ไม่ต้องแลกรหัส

เหมาะเมื่อทั้งสองเครื่องอยู่ **Personal Hotspot / LAN เดียวกัน** (วงที่ไม่มี Zscaler/firewall มากั้น) — แลกรหัสครั้งเดียวก็ไม่ต้อง แถมหลุดแล้วต่อกลับเอง · ขั้นตอนยุ่งอยู่ที่เครื่องหลักฝั่งเดียว

**เครื่องหลัก** (รัน broker — ทำครั้งเดียว):
```bash
cd WebBeam
node broker.mjs        # จะพิมพ์ IP ของเครื่องนี้ออกมา
```
จากนั้นเปิด `sync.html` → ในแผง "การเชื่อมต่อ" เลือก **เครื่องหลัก** · Broker IP = `localhost` · กด **เริ่มอัตโนมัติ** · เลือกโฟลเดอร์

**อีกเครื่อง** (กดไม่กี่ที):
เปิด `sync.html` → เลือก **อีกเครื่อง** · ใส่ Broker IP = IP ที่ broker พิมพ์ออกมา · กด **เริ่มอัตโนมัติ** · เลือกโฟลเดอร์ → เชื่อมเอง

- ติ๊ก **"จำไว้ — เปิดหน้ามาเชื่อมให้อัตโนมัติเลย"** แล้วครั้งหน้าเปิดหน้าก็ต่อเองไม่ต้องกดอะไร
- broker เป็นแค่ "ตู้ฝากรหัส" ตอนจับคู่ — **ไฟล์ยังวิ่ง P2P ตรงเหมือนเดิม ไม่ผ่าน broker** · ไม่มี dependency (Node ล้วน) ไม่ต้อง npm install
- ทดสอบ: `node test/broker-test.mjs`

## เงื่อนไข / ข้อจำกัด

- ทั้งสองเครื่องต้องอยู่ **Wi-Fi/LAN วงเดียวกัน** (ใช้ host/mDNS ต่อตรง ไม่ต้องมีเน็ต) · เครือข่ายต้องไม่เปิด client isolation
- กล้องสแกน QR + ปุ่มคัดลอก ต้องเปิดผ่าน **https** (เช่น URL ที่ deploy) หรือ localhost
- ไฟล์ที่รับเก็บใน memory ก่อน (เหมาะกับ source file/ไฟล์ทั่วไป) · ไฟล์ที่รับจะ "บันทึก" ได้ในเซสชันนั้น (ปิดหน้าแล้วต้องส่งใหม่) ส่วนไฟล์ข้อความเล็กจะอยู่ในประวัติถาวร
- ประวัติเก็บข้อความล่าสุด ~800 รายการ (localStorage จำกัด ~5MB)
