# WebBeam Chat

แชต + ส่งโค้ด/ไฟล์ ระหว่างเครื่องที่อยู่ **LAN เดียวกัน** แบบ peer-to-peer (WebRTC)
ทำงาน **offline** · **ไม่ต้องติดตั้ง** · **ไม่ผ่านเซิร์ฟเวอร์** · ประวัติเก็บในเครื่อง (local)

> ✅ ทดสอบจริงด้วย Chrome 2 instance: เชื่อมต่อ + แชต + ส่ง source file (โชว์ inline) + history persist หลัง reload (ดู [test/](test/))

## ใช้งาน

เปิดในเบราว์เซอร์ทั้งสองเครื่อง (แนะนำ Chrome/Edge)

1. ตั้ง **ชื่อของคุณ** → เลือกบทบาท **A (เริ่ม)** / **B (เข้าร่วม)**
2. แลกรหัสเชื่อมต่อกัน 1 รอบ (คัดลอก-วาง หรือ **สแกน QR**) → กดเชื่อมต่อ
3. จุดสถานะเป็น 🟢 → แผงเชื่อมต่อจะพับเอง เริ่มคุยได้เลย

## ฟีเจอร์

- **แชตข้อความ** — Enter ส่ง, Shift+Enter ขึ้นบรรทัด
- **ส่งเป็นบล็อกโค้ด** — ปุ่ม `</> โค้ด` (โชว์ monospace + ปุ่มคัดลอก)
- **ส่งไฟล์** — ปุ่ม 📎 หรือ **ลากมาวางในแชต**
  - **source file** (.swift/.js/.py/… ≤256KB) → โชว์เนื้อหา inline + คัดลอก/บันทึก
  - ไฟล์อื่น → การ์ดไฟล์ + ปุ่มบันทึก (ดาวน์โหลด)
- **ประวัติแชต (local)** — เก็บใน `localStorage` ของเบราว์เซอร์ (แต่ละเครื่องเก็บของตัวเอง) · เปิดใหม่ก็ยังอยู่ · ปุ่ม 🗑 ล้างได้

## 🔄 โหมดซิงค์โฟลเดอร์ ([sync.html](sync.html))

ซิงค์โฟลเดอร์ **สองทาง** ระหว่าง 2 เครื่องบน LAN (P2P, offline) — เหมาะกับโปรเจกต์ source code

- **เชื่อมด้วยรหัสห้อง** (PeerJS): ตั้งรหัสตรงกันทั้งสองเครื่อง → กดเชื่อมต่อ (เครื่องแรก=host, อีกเครื่อง=join อัตโนมัติ) · **จำรหัสไว้ ต่อใหม่เองหลัง refresh ไม่ต้อง pair ใหม่** · ตอนจับคู่ต้องมีเน็ต ส่งไฟล์ยัง P2P บน LAN · มีโหมด offline (แลกรหัส/QR เอง) เป็น fallback
- เลือกโฟลเดอร์ทั้งสองเครื่อง → auto-sync ทุก ~5 วิ (หรือกด Sync เอง)
- เทียบไฟล์ด้วย **hash** (กัน sync วนไม่จบ) · รองรับโฟลเดอร์ย่อย
- **ไม่ลบไฟล์ข้ามเครื่อง** (กันข้อมูลหาย) · ข้าม `.DS_Store`, `.git`, `node_modules`
- **แก้ชนกัน (conflict)** → เก็บทั้งสองไฟล์ (ตัวที่มาทับเปลี่ยนชื่อเป็น `ชื่อ.conflict-<เครื่อง>`)
- ✅ ทดสอบ engine จริง (in-memory store บน WebRTC): เพิ่มสองทาง + อัปเดต + conflict เก็บทั้งสอง + ไม่ ping-pong
- ข้อจำกัด: **Chrome/Edge เท่านั้น** · เบราว์เซอร์ poll หาความเปลี่ยนแปลง (near-realtime ~ทุกไม่กี่วิ ไม่ใช่ทันที) · ต้องกดอนุญาตโฟลเดอร์ใหม่ทุกครั้งที่เปิด

## เงื่อนไข / ข้อจำกัด

- ทั้งสองเครื่องต้องอยู่ **Wi-Fi/LAN วงเดียวกัน** (ใช้ host/mDNS ต่อตรง ไม่ต้องมีเน็ต) · เครือข่ายต้องไม่เปิด client isolation
- กล้องสแกน QR + ปุ่มคัดลอก ต้องเปิดผ่าน **https** (เช่น URL ที่ deploy) หรือ localhost
- ไฟล์ที่รับเก็บใน memory ก่อน (เหมาะกับ source file/ไฟล์ทั่วไป) · ไฟล์ที่รับจะ "บันทึก" ได้ในเซสชันนั้น (ปิดหน้าแล้วต้องส่งใหม่) ส่วนไฟล์ข้อความเล็กจะอยู่ในประวัติถาวร
- ประวัติเก็บข้อความล่าสุด ~800 รายการ (localStorage จำกัด ~5MB)
