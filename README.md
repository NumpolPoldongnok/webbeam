# WebBeam

**แชต + แนบไฟล์** ระหว่าง 2 เครื่องบน **LAN เดียวกัน** แบบ peer-to-peer (WebRTC) —
offline · ไม่ต้องติดตั้ง · ไฟล์วิ่งตรงระหว่างเครื่อง ไม่ผ่านเซิร์ฟเวอร์

แอปอยู่ที่ [sync.html](sync.html) — `index.html` จะ redirect มาที่นี่

> ✅ ทดสอบจริงด้วย Chrome 2 instance + headless test: เชื่อมต่อ + แชต + ส่งไฟล์ (text/binary/รูป) + history persist หลัง reload (ดู [test/](test/))

## ใช้งาน

เปิดในเบราว์เซอร์ทั้งสองเครื่อง (แนะนำ Chrome/Edge) แล้วเชื่อมต่อ 1 รอบ — มี 2 วิธี:

- 🔥 **Hotspot (อัตโนมัติ)** — รัน broker จิ๋วบนเครื่องหลักเครื่องเดียว ทั้งสองเครื่องจับคู่กันเอง หลุดแล้วต่อกลับเอง (ดูหัวข้อด้านล่าง)
- ✋ **Manual (แลกรหัส)** — เลือกบทบาท A/B → แลกรหัส (คัดลอก-วาง หรือ **สแกน QR**) → วางรหัสแล้วเชื่อมให้อัตโนมัติ

จุดสถานะเป็น 🟢 → แผงเชื่อมต่อพับเอง เริ่มแชตได้เลย

## ฟีเจอร์แชต

- **ข้อความ** — Enter ส่ง · Shift+Enter ขึ้นบรรทัด · ข้อความยาวมีปุ่มคัดลอกทั้งบนและล่าง
- **บล็อกโค้ด** — ปุ่ม `</> โค้ด` (โชว์ monospace + ปุ่มคัดลอก)
- **แนบไฟล์** — ปุ่ม 📎 · **ลากมาวาง** ในแชต · หรือ **วาง (paste) รูป/ไฟล์จาก clipboard** ในช่องพิมพ์ส่งได้เลย
  - **รูปภาพ** → แสดงพรีวิวในแชต คลิกเปิดเต็ม + ปุ่มบันทึก
  - **source file** (.swift/.js/.py/… ≤256KB) → โชว์เนื้อหา inline + คัดลอก/บันทึก
  - ไฟล์อื่น (zip/pdf/binary) → การ์ดไฟล์ + ขนาด + ปุ่มบันทึก (ดาวน์โหลด)
- **progress** ระหว่างส่ง/รับไฟล์ · ไฟล์ข้อความบีบ gzip อัตโนมัติตอนส่ง
- **ประวัติแชต (local)** — เก็บใน `localStorage` (แต่ละเครื่องของตัวเอง) · เปิดใหม่ก็ยังอยู่ · ปุ่ม 🗑️ ล้างได้
- **เครือข่ายบล็อก mDNS แล้วต่อไม่ติด?** ใส่ **"IP เครื่องนี้ (LAN)"** ทั้งสองเครื่อง (System Settings → Wi-Fi → Details) แล้วสร้างรหัสใหม่ — ใช้ IP จริงแทน `.local`
- ข้อจำกัด: **Chrome/Edge เท่านั้น**

> ใช้แชตนี้แนบ **zip** ส่งโปรเจกต์ทั้งก้อนได้ (zip เองที่เครื่อง → ลากเข้าแชต → อีกฝั่งกดบันทึกแล้วแตกเอง) — เรียบง่ายและเร็วกว่าให้แอปไล่เขียนไฟล์ทีละไฟล์

## 🔥 โหมด Hotspot — เชื่อมอัตโนมัติ ไม่ต้องแลกรหัส

เหมาะเมื่อทั้งสองเครื่องอยู่ **Personal Hotspot / LAN เดียวกัน** — แลกรหัสครั้งเดียวก็ไม่ต้อง หลุดแล้วต่อกลับเอง

**เครื่องหลัก** (รัน broker — ทำครั้งเดียว):
```bash
cd WebBeam
node broker.mjs        # จะพิมพ์ IP ของเครื่องนี้ออกมา
```
จากนั้นเปิด `sync.html` → แผง "การเชื่อมต่อ" เลือก **เครื่องหลัก** · Broker IP = `localhost` · กด **เริ่มอัตโนมัติ**

**อีกเครื่อง:** เปิด `sync.html` → เลือก **อีกเครื่อง** · ใส่ Broker IP = IP ที่ broker พิมพ์ออกมา · กด **เริ่มอัตโนมัติ** → เชื่อมเอง

- ติ๊ก **"จำไว้ — เปิดหน้ามาเชื่อมให้อัตโนมัติเลย"** ครั้งหน้าเปิดหน้าก็ต่อเอง
- broker เป็นแค่ "ตู้ฝากรหัส" ตอนจับคู่ — **ไฟล์ยังวิ่ง P2P ตรง ไม่ผ่าน broker** · ไม่มี dependency (Node ล้วน) ไม่ต้อง npm install
- ⚠️ ถ้าเปิดหน้าผ่าน **https** (เช่น github.io) browser จะบล็อก broker ที่เป็น http (mixed content) — เปิด `sync.html` แบบ **file://** หรืออนุญาต "Insecure content" ในการตั้งค่าไซต์
- ทดสอบ: `node test/broker-test.mjs`

## เงื่อนไข / ข้อจำกัด

- ทั้งสองเครื่องต้องอยู่ **Wi-Fi/LAN วงเดียวกัน** · เครือข่ายต้องไม่เปิด client isolation
- กล้องสแกน QR + ปุ่มคัดลอก ต้องเปิดผ่าน **https** หรือ localhost/file
- ไฟล์ที่รับเก็บใน memory · กด "บันทึก" ได้ในเซสชันนั้น (ปิด/รีโหลดหน้าแล้วต้องส่งใหม่) ส่วนไฟล์ข้อความเล็กจะอยู่ในประวัติถาวร
- ประวัติเก็บ ~500 รายการล่าสุด (localStorage จำกัด ~5MB)
