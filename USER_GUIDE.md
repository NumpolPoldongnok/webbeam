# คู่มือใช้งาน WebBeam

ส่งไฟล์ระหว่างเครื่องที่อยู่ใกล้กัน ผ่านเบราว์เซอร์ ไม่ต้องติดตั้ง ไม่ต้องมี internet
(ขอแค่ทั้งสองเครื่องอยู่ Wi-Fi/LAN วงเดียวกัน)

---

## ⚙️ เตรียมก่อนเริ่ม

1. **ทั้งสองเครื่องต่อ Wi-Fi/LAN วงเดียวกัน** — ไม่ต้องมี internet ก็ได้ แต่ต้องวงเดียวกัน
2. **แนะนำใช้ Chrome หรือ Edge ทั้งสองเครื่อง** (Safari รับไฟล์ใหญ่ไม่ได้)
3. เปิดแอป: ดับเบิลคลิก `index.html` หรือเปิด URL ที่โฮสต์ไว้ — **เปิดทั้งสองเครื่อง**

---

## 🔗 ขั้นที่ 1–2 · เชื่อมต่อกัน (ทำครั้งเดียว)

ต้องแลก "รหัสเชื่อมต่อ" กัน 1 รอบ คิดง่ายๆ คือ **A ทัก → B ตอบ → A รับสาย**

### เครื่อง A (ผู้เริ่ม)
1. กดบทบาท **"เริ่มการเชื่อมต่อ (เครื่อง A)"**
2. กด **"สร้างรหัสของฉัน"** → กด **"คัดลอก"** → ส่งรหัสให้เครื่อง B
3. รอรหัสตอบกลับจาก B แล้ววางในช่อง → กด **"เชื่อมต่อ"**

### เครื่อง B (ผู้เข้าร่วม)
1. กดบทบาท **"เข้าร่วม (เครื่อง B)"**
2. วางรหัสจาก A ในช่อง → กด **"สร้างรหัสตอบกลับ"**
3. กด **"คัดลอก"** → ส่งรหัสตอบกลับไปให้ A

> **วิธีแลกรหัส:** ส่งผ่าน LINE / Messages / โน้ตที่แชร์กัน หรือถ้าเครื่องอยู่ติดกันพิมพ์ตามก็ได้
> (รหัสยาวหน่อย แนะนำคัดลอก-วาง)

### ดูจุดสถานะ (วงกลมเล็กๆ)
- ⚪️ เทา = ยังไม่เชื่อมต่อ
- 🟠 ส้ม = กำลังเชื่อมต่อ
- 🟢 **เขียว = เชื่อมต่อแล้ว พร้อมส่ง** ← รอให้ขึ้นสีนี้

---

## 📥 ขั้นที่ 3 · ตั้งโฟลเดอร์รับไฟล์ (ฝั่งที่จะรับ)

ในการ์ด **"โฟลเดอร์ปลายทาง"** → กด **"เลือกโฟลเดอร์…"** → เลือกที่เก็บ (เช่น Downloads)
- ✅ เลือกแล้ว = ไฟล์ใหญ่ stream ลงดิสก์ตรงๆ รับหลาย GB ได้
- ไม่เลือก = ไฟล์เด้งดาวน์โหลดผ่านเบราว์เซอร์แทน (จำกัดขนาดตามหน่วยความจำ)

---

## 📤 ขั้นที่ 4 · ส่งไฟล์

เมื่อจุดสถานะเป็น 🟢 กรอบประ "ลากไฟล์มาวาง" จะใช้งานได้:
- **ลากไฟล์มาวาง** ในกรอบ (หลายไฟล์พร้อมกันได้) หรือ **คลิกที่กรอบ** เพื่อเลือกไฟล์
- เห็น **แถบ progress + ความเร็ว (MB/s)** ของแต่ละไฟล์ · ขึ้น "เสร็จ ✓" เมื่อครบ
- **ส่งได้สองทาง** — เครื่องไหนลากไฟล์ เครื่องนั้นเป็นผู้ส่ง อีกฝั่งรับ

---

## 🛠️ แก้ปัญหาที่พบบ่อย

| อาการ | สาเหตุ / วิธีแก้ |
|---|---|
| จุดสถานะไม่ขึ้นเขียวสักที | คนละ Wi-Fi กัน หรือเครือข่ายเปิด **client isolation** (Wi-Fi สาธารณะ คาเฟ่ โรงแรม) → ใช้ Wi-Fi บ้าน หรือเปิด hotspot เครื่องนึงให้อีกเครื่องเกาะ |
| ปุ่ม "คัดลอก" / "เลือกโฟลเดอร์" กดไม่ได้ | เปิดผ่าน `http://` ธรรมดา → ต้องเป็น `file://`, `localhost` หรือ `https://` (เลือก-คัดลอกข้อความเองได้) |
| รับไฟล์ใหญ่แล้วเบราว์เซอร์ค้าง | ใช้ Safari อยู่ → เปลี่ยนเป็น **Chrome/Edge** แล้วกด "เลือกโฟลเดอร์" ก่อนรับ |
| ขึ้น "รหัสไม่ถูกต้อง" | คัดลอกรหัสไม่ครบ → เลือกทั้งช่องแล้วคัดลอกใหม่ |
| ส่งเสร็จแล้วอยากส่งอีก | ลากไฟล์เพิ่มได้เลย ไม่ต้องเชื่อมต่อใหม่ (ตราบใดที่ยังเขียว) |

---

## ❓ คำถามที่พบบ่อย

**ปลอดภัยไหม?** — ข้อมูลส่งแบบเข้ารหัส (DTLS) และวิ่งตรงเครื่องต่อเครื่องบน LAN ไม่ผ่านเซิร์ฟเวอร์

**ส่งโฟลเดอร์ได้ไหม?** — ส่งได้เฉพาะไฟล์ ถ้าจะส่งทั้งโฟลเดอร์ให้ zip ก่อน

**ส่งข้ามมือถือ/Windows ได้ไหม?** — ได้ ขอแค่เปิดในเบราว์เซอร์ที่รองรับและอยู่ LAN เดียวกัน
